# ExampleAI documentation — link it instead of paraphrasing

> EDIT ME. This is an example skill shipped by your customization repo. It
> fills the gap a white-label agent has by design: the platform does NOT
> ship its own docs skill to your agent, so your agent has no idea where
> YOUR docs live until you tell it here. Replace ExampleAI / the URLs below
> with your brand, then keep this file in `skills/product-docs/`.

The canonical user-facing docs live at **`https://docs.example.com/`**. When
the user asks *how the product works*, *how files are handled*, *which AI
provider to pick*, or *what use cases it's best at*, **link them to the
relevant page rather than paraphrasing** — the docs are authoritative and
stay in sync, while a paraphrase drifts.

Pages worth knowing (edit to match your real docs):

- [`/getting-started/`](https://docs.example.com/getting-started/) — first
  steps, what the agent can do, and how to drive it from chat.
- [`/billing/`](https://docs.example.com/billing/) — how credit, top-ups,
  and renewals work for your customers.
- [`/privacy/`](https://docs.example.com/privacy/) — where uploads live and
  how files are handled. Cite this for privacy questions.

Match the user's chat language when there's a clear locale and your docs
have a translation for it.
